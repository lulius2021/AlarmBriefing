5. Bot Access (API)

Users can generate an API key allowing an external bot (e.g. Clawdbot) to access their data. Access is limited to defined scopes:

  • alarms:read/write — Read and create alarms
  • briefings:write — Generate briefings
  • settings:read — Read settings

Users can revoke API keys at any time. Bot actions are logged in the audit log.

6. Your Rights

Under the GDPR, you have the right to:

  • Access (Art. 15) — What data we have about you
  • Rectification (Art. 16) — Correct inaccurate data
  • Erasure (Art. 17) — Delete your account and all data
  • Portability (Art. 20) — Export your data
  • Objection (Art. 21) — Object to processing
  • Complaint — To the competent data protection authority

7. Account Deletion

You can permanently delete your account at any time in the app under Settings → "Delete account". All data is irrevocably removed:

  • Account data (email, name, password)
  • All alarm settings
  • All API keys
  • All briefings
  • Audit logs

Alternatively via email to julius.deusch@trainq.app.

8. Cookies & Tracking

AlarmBriefing uses no cookies and no tracking. We use localStorage in the browser solely for session data (auth token). No analytics or advertising trackers are used.

9. Data Security

  • Passwords are hashed with bcrypt (12 rounds)
  • API keys are stored as SHA-256 hashes
  • All connections use HTTPS/TLS
  • JWT tokens expire after 30 days

10. Changes

We reserve the right to update this privacy policy. For significant changes, we will notify you via email or in-app notice.
